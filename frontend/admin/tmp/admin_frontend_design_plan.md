# 管理用フロントエンドサーバ 設計・実装計画

## 1. 概要

MFG ドローン自動追従撮影システムの管理者用フロントエンドサーバの詳細設計および実装計画です。

### 1.1 システム要件
- **技術スタック**: Python 3.11 + Flask
- **動作環境**: Windows 11 Pro 64bit
- **バックエンドAPI**: Raspberry Pi 5 (http://192.168.1.100:8000)
- **主要機能**: 物体認識モデル訓練、ドローン制御、追跡管理

## 2. 機能要件

### 2.1 物体認識モデル訓練機能

#### 2.1.1 画像アップロード機能
- **目的**: 学習用画像をアップロードしモデル訓練を実行
- **UI要素**:
  - ファイル選択（複数選択対応）
  - オブジェクト名入力フィールド
  - アップロード進捗表示
- **API連携**: `/model/train` (POST, multipart/form-data)

#### 2.1.2 モデル管理機能
- **目的**: 訓練済みモデルの一覧表示と管理
- **UI要素**:
  - モデル一覧テーブル（名前、作成日時、精度）
  - モデル削除機能
- **API連携**: `/model/list` (GET)

### 2.2 ドローン制御機能

#### 2.2.1 接続管理
- **目的**: ドローンとの接続状態管理
- **UI要素**:
  - 接続状態表示
  - 接続/切断ボタン
- **API連携**: `/drone/connect`, `/drone/disconnect` (POST)

#### 2.2.2 基本制御
- **目的**: ドローンの基本操作（離陸、着陸、緊急停止）
- **UI要素**:
  - 離陸ボタン
  - 着陸ボタン
  - 緊急停止ボタン
- **API連携**: `/drone/takeoff`, `/drone/land`, `/drone/emergency` (POST)

#### 2.2.3 手動操縦
- **目的**: 管理者による手動ドローン操縦
- **UI要素**:
  - 方向キー（上下左右前後）
  - 回転制御（時計回り、反時計回り）
  - 距離・角度入力
- **API連携**: `/drone/move`, `/drone/rotate` (POST)

### 2.3 追跡制御機能

#### 2.3.1 追跡開始/停止
- **目的**: 物体追跡の開始・停止制御
- **UI要素**:
  - 対象オブジェクト選択（ドロップダウン）
  - 追跡モード選択（center/follow）
  - 開始/停止ボタン
- **API連携**: `/tracking/start`, `/tracking/stop` (POST)

#### 2.3.2 追跡状態監視
- **目的**: 追跡状態のリアルタイム監視
- **UI要素**:
  - 追跡状態表示
  - 対象物検出状況
  - 対象物位置情報
- **API連携**: `/tracking/status` (GET, 定期ポーリング)

### 2.4 センサー・状態監視機能

#### 2.4.1 ドローン状態表示
- **目的**: ドローンの状態をリアルタイム表示
- **UI要素**:
  - バッテリー残量表示
  - 飛行高度表示
  - 温度表示
  - 姿勢角表示
- **API連携**: `/drone/status`, `/drone/battery` 等 (GET, 定期ポーリング)

#### 2.4.2 カメラプレビュー
- **目的**: ドローンからの映像をリアルタイム表示
- **UI要素**:
  - ビデオストリーム表示エリア
- **API連携**: `/camera/stream` (GET, ストリーミング)

## 3. 技術設計

### 3.1 アーキテクチャ

```
[ブラウザ] ←→ [Flask Webアプリ] ←→ [バックエンドAPI]
    ↑              ↑                    ↑
 WebSocket    Socket.IO          HTTP/REST API
  (リアルタイム)  (リアルタイム)      (Raspberry Pi)
```

### 3.2 主要技術コンポーネント

#### 3.2.1 Flask アプリケーション構造
```
frontend/admin/
├── app.py                 # メインアプリケーション
├── config.py             # 設定管理
├── blueprints/           # ルーティング分割
│   ├── __init__.py
│   ├── drone_control.py  # ドローン制御関連
│   ├── model_training.py # モデル訓練関連
│   └── tracking.py       # 追跡制御関連
├── services/             # バックエンドAPI連携
│   ├── __init__.py
│   ├── api_client.py     # HTTP API クライアント
│   └── websocket_client.py # WebSocket/Socket.IO
├── templates/            # Jinja2テンプレート
├── static/              # 静的ファイル
│   ├── css/
│   ├── js/
│   └── images/
└── utils/               # ユーティリティ
```

#### 3.2.2 リアルタイム通信
- **Flask-SocketIO**: サーバ・クライアント間のリアルタイム通信
- **定期ポーリング**: センサーデータ取得（1秒間隔）
- **WebSocket**: 追跡状態更新の即座な反映

#### 3.2.3 APIクライアント設計
```python
class DroneAPIClient:
    def __init__(self, base_url: str):
        self.base_url = base_url
        
    # 接続管理
    async def connect_drone(self) -> dict
    async def disconnect_drone(self) -> dict
    
    # 制御コマンド
    async def takeoff(self) -> dict
    async def land(self) -> dict
    async def move(self, direction: str, distance: int) -> dict
    
    # 追跡制御
    async def start_tracking(self, target: str, mode: str) -> dict
    async def stop_tracking(self) -> dict
    
    # モデル管理
    async def train_model(self, object_name: str, images: list) -> dict
    async def list_models(self) -> dict
```

- **Webサーバ**: Gunicorn + Nginx (推奨)
- **セキュリティ**: HTTPS、CSRF保護
- **ログ管理**: ファイルベースログ

## 7. リスク・課題

### 7.1 技術的リスク
- **ネットワーク遅延**: WiFiネットワークでの通信品質
- **リアルタイム性**: ストリーミング・制御コマンドの遅延
- **モデル訓練時間**: 大量画像処理時の応答性

### 7.2 対策
- **タイムアウト設定**: API呼び出しのタイムアウト適切設定
- **非同期処理**: 重い処理の非同期実行
- **キャッシュ**: 頻繁にアクセスするデータのキャッシュ
