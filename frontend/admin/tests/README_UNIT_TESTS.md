# 管理用フロントエンドサーバ単体テスト

## 概要

このディレクトリには、`frontend/admin` の Flask アプリケーション（`main.py`）に対する包括的な単体テストが含まれています。これらのテストは、ドローン実機やバックエンドサーバーを必要とせず、独立して実行できます。

## テスト構成

### 📁 ディレクトリ構造

```
tests/
├── mocks/                          # モック実装
│   ├── __init__.py
│   └── backend_mock.py             # バックエンドAPIモック
├── fixtures/                       # テストフィクスチャ
│   ├── __init__.py
│   └── unit_test_fixtures.py       # 共通フィクスチャとテストデータ
├── test_design.md                  # テスト設計書
├── test_drone_api_client.py        # DroneAPIClient基本テスト
├── test_drone_api_client_advanced.py # DroneAPIClient高度機能テスト
├── test_flask_routes.py            # Flask ルート基本テスト
├── test_flask_routes_advanced.py   # Flask ルート高度機能テスト
└── README_UNIT_TESTS.md           # このファイル
```

## テスト方針

### ✅ 正常系テスト
- **境界値テスト**: 引数の最小値・最大値での動作確認

- **デフォルト値テスト**: オプション引数のデフォルト値テスト

### ❌ 異常系テスト
- **範囲外値テスト**: 引数の範囲外値でのエラーハンドリング確認
- **不正型テスト**: 不正な型の引数でのエラーハンドリング確認
- **None値テスト**: None値での動作確認

### ⚠️ エラーハンドリングテスト
- **ネットワークエラー**: requests.exceptions のシミュレーション
- **HTTPエラー**: 各種HTTPステータスコード（4xx, 5xx）の処理確認
- **タイムアウトエラー**: 接続タイムアウト・読み取りタイムアウトの処理確認

## 実行方法

### 📋 前提条件
```bash
# 依存関係のインストール
pip install -r test_requirements.txt
```

### 🚀 実行コマンド

#### 1. **専用スクリプトでの実行（推奨）**
```bash
# 全テスト実行
python run_unit_tests.py

# 特定タイプのみ実行
python run_unit_tests.py --type api_client    # DroneAPIClient のみ
python run_unit_tests.py --type flask_routes  # Flask ルートのみ
python run_unit_tests.py --type basic         # 基本機能のみ
python run_unit_tests.py --type advanced      # 高度機能のみ

# 詳細出力・カバレッジ無効化
python run_unit_tests.py --verbose --no-coverage
```

#### 2. **pytest での直接実行**
```bash
# 単体テスト用設定での実行
pytest -c pytest_unit.ini -v

# カバレッジ付きで実行
pytest -c pytest_unit.ini --cov=main --cov-report=html

# 特定ファイルのみ実行
pytest tests/test_drone_api_client.py -v
```

### 📊 レポート出力
- **HTMLレポート**: `tests/reports/unit_test_report.html`
- **カバレッジレポート**: `tests/reports/coverage_html/index.html`
- **JSONレポート**: `tests/reports/unit_test_report.json`

## トラブルシューティング

### ❓ よくある問題と解決方法

#### ImportError が発生する場合
```bash
# プロジェクトルートから実行していることを確認
cd /path/to/frontend/admin
python run_unit_tests.py
```

#### カバレッジが低い場合
```bash
# 詳細なカバレッジレポートで不足箇所を確認
pytest -c pytest_unit.ini --cov=main --cov-report=term-missing
```

#### テストが失敗する場合
```bash
# 詳細出力で失敗原因を確認
python run_unit_tests.py --verbose
```
